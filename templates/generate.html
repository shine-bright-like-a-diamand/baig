About model

The search for inhibitors has several features. First, since our goal is to create new molecules that may not exist in the database, the problem involves unsupervised learning. Second, there is no finite set of answers, that is, only the desired properties of the finite compounds exist. Models using autoencoder architecture are well suited for solving such problems.

Let us note the key features of the architecture of the specific neural network used in this work. There are three main parts used during model training: the encoder, the latent space representation of the molecule, and the decoder. Input and output molecules are presented in SMILES string format, and input data must also include molecular weight, lipophilicity, and total polar surface area (TPSA) values. When using a trained model, there are only two parts involved: the latent space representation of the data and the decoder. This means that only the molecular weight, lipophilicity and total polar surface area (TPSA) values ​​need to be entered. In latent space, a molecule is encoded with 128 numbers, each of which has a specific range. The decoder then reconstructs the molecule from the input data. The output format is SMILES strings containing descriptions of potential BCR-ABL tyrosine kinase inhibitors.
